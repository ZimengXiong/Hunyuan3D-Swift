## 3. (c) REFUTED / NUANCED — Code Contradicts or Complicates

1. **scale_factor divide location** (nuance, 3 verifiers): The plan says "divide latents by scale_factor *before post_kl, inside VAE decode()*." **The divide is NOT in `ShapeVAE.decode()/forward()`** — it's in the pipeline at `pipelines.py:670`, before `vae()`. Behaviorally correct, wrong location. **Fix**: add `latents = latents / scale_factor` at the top of the MLX decode entry (mirroring `pipelines.py:670`), not inside the transformer.

2. **`pe=None` line citation** (nuance): Plan cites `hunyuan3ddit.py:391`. Correct location is **`:393` in the Hunyuan3D-2.1 tree** (and `:413` in the cbun MLX copy — the two files are byte-identical except a 20-line license-header offset). `:391` matches **neither** file's `pe=None`. Substance (no RoPE) is correct.

3. **runtime.py "bf16→fp16 guard"** (nuance): Plan's parenthetical implies silent coercion. Reality: `runtime.py:86-87` **raises `ValueError`** on explicit bf16+MPS; only `dtype='auto'` defaults to fp16 (`:68-69`). Reword to "fp16 default + bf16-on-MPS reject." Also: **runtime.py is torch-typed throughout** (imports torch `:20`, returns `torch.device`/`torch.dtype`) — "reuse as-is" reintroduces torch into the helper layer, contradicting the "zero torch in inference path" goal unless reworked.

4. **MC level** (nuance): Not hardcoded to 0 — it's a runtime arg. FlowMatching path defaults `mc_level=0.0` (`pipelines.py:704`) — correct for the target — but the legacy pipeline defaults `-1/512` (`:571`). Pass it through, default 0.0.

5. **mesh_simplifier.bin "x86/CUDA"** (unverifiable): The binary is genuinely **absent** from disk (verified). Its x86/CUDA nature is an upstream assertion that **cannot be confirmed here** — the only verifiable fact is absence. The mitigation (use FaceReducer) is sound regardless.

6. **cbun Euler loop "reusable as-is"** (nuance): Loop *math* is pure MLX, but setup/teardown bridges through torch+numpy (`:61-72`), and cbun's schedule is **SiT transport** (`transport.check_interval`, plain `linspace(t0,t1)` with **no shift warp**) — a different integrator family than FlowMatchEuler. Plan's instinct to re-derive the sigma schedule is correct.

7. **sigma schedule "linspace warped by shift"** (nuance): Two-part. Pipeline builds `linspace(0,1)` (`pipelines.py:738`) and passes it as `sigmas=`, which makes `set_timesteps` **skip** its internal `linspace(sigma_max→sigma_min)` branch (`schedulers.py:201`). Then the **rational** warp `shift*x/(1+(shift-1)*x)` (`:212`) — NOT the exp-based `time_shift` (which only fires under `use_dynamic_shifting=True` + `mu`). A trailing `1.0` is appended to sigmas (`:218`).

8. **rembg in pipeline** (nuance): Not inside `__call__`; standalone upstream step (`rembg.py`). Plan's diagram shows this correctly.

## 4. RANKED RISK — Unverified Assumptions That Would Silently Corrupt the Mesh

Ranked by (parity impact × likelihood of being wrong). "Silent" = no crash, just wrong geometry.

1. **🔴 scale_factor value (1.0188...)** — Wrong value silently scales the entire SDF field → systematically wrong surface (bloated/shrunk mesh, no error). The plan's literal is NOT on disk, and the nearby `1.0039...` is a *different quantity* that's easy to grab by mistake. **Highest risk: wrong number AND a confusable decoy on disk.**

2. **🔴 giant-vs-large + SwiGLU FFN** — If 2mini is giant but you port large (or vice versa), every conditioning token is wrong → wrong cross-attention → wrong mesh, no crash if dims happen to align via config. Cascades into context_dim/text_len/DiT cross-attn dims and whether a SwiGLU port is even needed. On-disk evidence points to **large**, plan claims **giant** — direct conflict.

3. **🔴 ctx_in / context_in_dim (1536 vs 1024)** — Tied to #2. If wrong, the DiT `txt_in`/cross-attn projection is mis-shaped. Default is 1536 but on-disk mini denoiser uses 1024. A silent mismatch if the loader doesn't assert against weight shapes.

4. **🟠 ln_post eps (1e-5 vs 1e-6)** — Confirmed bug in cbun (`mlx_shape_vae.py:225` uses 1e-6; torch uses 1e-5). Small but real numerical drift on the final SDF projection. Easy to miss because it's a default-argument trap. **Fix is known and concrete.**

5. **🟠 qk_norm enabled for VAE (+ include_pi)** — cbun unconditionally reads q_norm/k_norm weights and sets include_pi=False; torch defaults are qk_norm=False / include_pi=True. If 2mini's actual flags differ from what you assume, you either read missing weights (crash) or skip a norm (silent drift). **Two independent flag mismatches.**

6. **🟠 sigma schedule shift value** — If `shift≠1.0` and you assume identity (or copy cbun's un-warped transport linspace), the noise schedule is wrong → wrong denoising trajectory → wrong latent → wrong mesh. Magnitude depends on actual shift.

7. **🟡 normalization stats (ImageNet vs 0.5/0.5)** — RESOLVED as a decoy: encoder hardcodes ImageNet internally. Risk only materializes if a porter naively copies the `0.5/0.5` from the training config. Low risk now that it's flagged, but high impact if mishandled.

8. **🟡 block counts 8+16 vs 16+32** — If wrong, weight loading shape-mismatches → likely a **loud** failure (crash), not silent corruption. Lower parity risk because it won't pass silently — but still must come from the real config.

9. **🟡 attention layout (Linear [out,in], Conv NCHW→NHWC)** — Confirmed correct; only DINO patch conv transposes. Low residual risk.
